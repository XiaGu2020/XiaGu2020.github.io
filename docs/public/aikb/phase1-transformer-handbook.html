1Tokenizer 原理与 BPE

1.1 五句话原理

  1. 计算机不懂文字，只懂数字 → 必须把文本切成 token 再映射成 id
  2. 切到字符太碎（"unhappy" → u/n/h/a/p/p/y 失语义），切到词太大（OOV 严重，词表爆炸）
  3. BPE（Byte-Pair Encoding）= 频次驱动的子词切分，平衡词表大小和语义
  4. 训练时统计 byte 对频次，反复合并最高频对到词表里直到达到目标大小
  5. 2024 后主流 LLM 都用 BPE 变体（GPT 用 cl100k_base，Qwen 用 tiktoken 兼容版）

1.2 BPE 算法（伪代码）

def bpe_train(corpus, vocab_size):
    # ① 初始词表 = 所有单字符
    vocab = set(c for word in corpus for c in word)
    splits = {w: list(w) for w in corpus}

    while len(vocab) < vocab_size:
        # ② 统计相邻 token 对的频次
        pair_freq = Counter()
        for word, sym_list in splits.items():
            for i in range(len(sym_list)-1):
                pair_freq[(sym_list[i], sym_list[i+1])] += corpus[word]

        # ③ 合并最高频对
        best = max(pair_freq, key=pair_freq.get)
        new_token = "".join(best)
        vocab.add(new_token)

        # ④ 在所有 splits 里把这个对替换为新 token
        for word in splits:
            splits[word] = merge_pair(splits[word], best, new_token)
    return vocab

⚠️ 易混淆
  • BPE vs WordPiece：BPE 选频次最高对，WordPiece（BERT 用）选最大化语料概率的对。结果差不多
  • SentencePiece：是 Google 的 tokenizer 库，可以选 BPE 或 Unigram 算法，跨语言友好
  • Tokenizer 是先训练好的：训练时已固定，推理时不能再改；微调小模型时通常沿用原 tokenizer

2Embedding：从词到向量

2.1 五句话原理

  1. token id 只是一个整数，模型没法直接计算 → 必须查表变成稠密向量
  2. Embedding 矩阵 = [vocab_size × d_model]，每行是一个 token 的向量表示
  3. 这个矩阵是可训练参数，训练时与其他权重一起反向传播更新
  4. 训练完成后，语义相近的 token 在向量空间里距离更近（"国王-男+女≈女王" 经典）
  5. 注意：模型内部的 token embedding ≠ 我们 RAG 用的 sentence embedding（前者按 token，后者按句子聚合）

2.2 Embedding lookup 在 PyTorch 里就一行

import torch
import torch.nn as nn

vocab_size, d_model = 50000, 768
emb = nn.Embedding(vocab_size, d_model)   # 等价于一张可训练大表

token_ids = torch.tensor([[101, 2034, 9089, 102]])   # [batch=1, seq=4]
x = emb(token_ids)                      # [1, 4, 768]
print(x.shape)

3Self-Attention 完整推导（最重要的一章）

3.1 直觉先行：注意力解决什么

RNN 处理"这只动物没过马路因为它太累了"时，"它"指什么？RNN 必须把信息逐步传递，距离越远越易丢失。注意力机制让序列里任意两个位置直接对话，根据"相关性"加权聚合信息——"它"可以同时看所有词，发现"动物"权重最大。

3.2 Q / K / V 三个角色

这是最被问烂的题——但正确答案其实只有一句："QKV 是同一序列的三个不同投影，分别扮演问询者、被检索者、信息提供者"。

角色类比图书馆来源
Query (Q)"我想找什么"（搜索关键词）x · W_Q
Key (K)"我是关于什么"（书的标签）x · W_K
Value (V)"我有什么内容"（书的正文）x · W_V

3.3 完整公式（背下来）

Attention(Q, K, V) = softmax( Q · KT / √d_k ) · V

3.4 五步分解（每一步都要能解释）

输入 X：[seq_len, d_model] 例如 (4, 768) Step 1 线性投影 Q = X @ W_Q # [4, 64] K = X @ W_K # [4, 64] V = X @ W_V # [4, 64] Step 2 打分（Q 和 K 内积） scores = Q @ K.T # [4, 4] 含义：第 i 行第 j 列 = 第 i 个 token 对第 j 个 token 的"相关度" Step 3 缩放（除根号 d_k） scores = scores / sqrt(64) 原因：d_k 越大，内积方差越大，softmax 变陡峭，梯度消失 Step 4 归一化（softmax） weights = softmax(scores, dim=-1) # [4, 4]，每行和为 1 含义：每个 token 该把多少注意力分给其他 token Step 5 加权求和 output = weights @ V # [4, 64] 含义：用 attention 权重对 V 做加权汇总

3.5 为什么除以 √d_k？（高频面试题）

设 q、k 的每一维都是均值 0、方差 1 的独立随机变量，则 q·k 的方差是 d_k。当 d_k 大（如 64、128）时，scores 的标准差 √d_k 也大，softmax 把绝大部分概率压到最大那一项 → 其他位置梯度几乎为 0，模型学不动。除以 √d_k 把方差拉回 1，softmax 平缓，梯度健康。

⚠️ 高频面试坑：
  • 问"为什么除 √d_k 而不是 √d_model"——答：每个 head 的维度是 d_k = d_model / h，是 head 维度而非整体
  • 问"K 和 V 必须不同吗"——答：理论上可同源（用 X 直接当 V），分开是给模型更多表达自由度，类似 ResNet 中的多分支

3.6 Causal Mask（因果遮罩，GPT 必备）

训练 Decoder-Only 模型时，预测第 i 个 token 不能偷看 i+1 之后的 token，否则就是抄答案。Mask 就是把 scores 上三角设为 -∞，softmax 后变成 0。

seq = 4
mask = torch.triu(torch.ones(seq, seq), diagonal=1).bool()
# mask = [[F, T, T, T],
#         [F, F, T, T],
#         [F, F, F, T],
#         [F, F, F, F]]
scores = scores.masked_fill(mask, float("-inf"))
4Multi-Head Attention

4.1 为什么要多头

一个注意力头只能学一种"视角"的依赖关系。多头 = 把 d_model 切成 h 份，每份各自做一次 attention，然后拼起来过一个线性层。类比：Java 里同一个对象用不同 interface 看到不同侧面。

4.2 维度变化（容易绕晕，画清楚）

d_model = 768, num_heads = 12, d_k = 768/12 = 64 输入 X: [batch, seq, 768] │ W_Q, W_K, W_V │ 各 [768, 768] ▼ Q,K,V: [batch, seq, 768] │ reshape 成多头 ▼ Q,K,V (multi-head): [batch, 12, seq, 64] # 把 768 切成 12 份每份 64 │ 每个 head 各自算 Attention ▼ heads_out: [batch, 12, seq, 64] │ concat 回去 ▼ out: [batch, seq, 768] │ W_O [768, 768] ▼ final: [batch, seq, 768]

4.3 参数量计算（面试可能让你算）

单层 Multi-Head Attention 参数量 = 4 × d_model² （W_Q, W_K, W_V, W_O 各一个）。
例：d_model=768 → 4 × 768² ≈ 2.36M 参数/层。一个 12 层 BERT-Base 仅 attention 部分就 28M 参数。

5手写 10 行 Attention（必须自己敲一遍）

5.1 极简版（10 行核心）

import torch
import torch.nn.functional as F
import math

def attention(Q, K, V, mask=None):
    d_k = Q.size(-1)
    scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))
    weights = F.softmax(scores, dim=-1)
    return weights @ V, weights

5.2 带多头的完整模块（30 行）

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=768, num_heads=12):
        super().__init__()
        assert d_model % num_heads == 0
        self.h = num_heads
        self.d_k = d_model // num_heads
        self.W_Q = nn.Linear(d_model, d_model)
        self.W_K = nn.Linear(d_model, d_model)
        self.W_V = nn.Linear(d_model, d_model)
        self.W_O = nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        B, T, D = x.shape
        # 投影 + reshape 成 [B, h, T, d_k]
        Q = self.W_Q(x).view(B, T, self.h, self.d_k).transpose(1,2)
        K = self.W_K(x).view(B, T, self.h, self.d_k).transpose(1,2)
        V = self.W_V(x).view(B, T, self.h, self.d_k).transpose(1,2)

        scores = Q @ K.transpose(-2,-1) / math.sqrt(self.d_k)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float("-inf"))
        attn = F.softmax(scores, dim=-1)
        out = attn @ V                        # [B, h, T, d_k]

        # concat 回去 + 输出投影
        out = out.transpose(1,2).contiguous().view(B, T, D)
        return self.W_O(out)
验证：跑这段不报错就 OK。
mha = MultiHeadAttention()
x = torch.randn(2, 10, 768)
print(mha(x).shape) # torch.Size([2, 10, 768])

7位置编码（重点 RoPE）

7.1 为什么需要位置编码

Self-Attention 是 位置无关 的——把 "我打你" 和 "你打我" 的输入顺序打乱，attention 输出也打乱但内容相同，模型分不清谁打谁。所以必须额外注入位置信息。

7.2 三代位置编码演进

代方法典型模型问题
1代固定正弦余弦（绝对位置）原版 Transformer外推差
2代可学习位置 embeddingBERT, GPT-2超过训练长度直接坏
3代相对位置（T5 bias / ALiBi / RoPE）2024 主流各有取舍

7.3 RoPE（Rotary Position Embedding）核心

2024 后几乎所有主流 LLM（Llama / Qwen / DeepSeek / GLM）都用 RoPE。一句话：用旋转矩阵把位置信息编进 Q 和 K。

关键性质：

  • 相对位置自然嵌入：q_m 和 k_n 的内积只依赖 m-n 的差值
  • 训练 4K 推理 32K 也能用（外推性好，配合 NTK / YaRN 可达 128K）
  • 计算开销小，不增加参数量
RoPE: q_m' = R_θ_m · q_m, k_n' = R_θ_n · k_n
<q_m', k_n'> = <q_m, k_n>_θ_(m-n)
⚠️ 面试常被深问：上下文窗口怎么从 4K 扩到 128K？
答：① RoPE base 频率插值（PI）② NTK-aware（高频不变，低频拉伸）③ YaRN（更智能的频率分段缩放）④ 长度外推训练（先短后长继续训练）。这是 Qwen2.5、Llama3 长上下文的核心技术。

8.2 Pre-Norm vs Post-Norm（细节但常考）

Post-Norm（原版 2017） Pre-Norm（GPT-2 起主流） ▲ ▲ LayerNorm + (残差) │ ┌──┴──┐ + (残差) │ │ ┌─┴─┐ FFN LayerNorm │ │ │ │ FFN │ └──┬──┘ │ │ ▲ └─┬─┘ │ │ ... ...

Pre-Norm 优势：训练更稳定，深层模型不易梯度爆炸。代价：表达能力略弱。2024 后几乎全用 Pre-Norm，且 Norm 多换成 RMSNorm（Llama 用，速度更快）。

8.3 残差连接（Skip Connection）的作用

  • 梯度高速公路：让梯度可以直接从顶层流到底层，缓解深层网络梯度消失
  • 恒等学习：当前层最差也能学成"什么都不变"，不会比浅层差
  • 是堆深层的前提：没有残差，10 层以上的 Transformer 训不动

9.2 你为什么仍然用 RoBERTa 做审核

RoBERTa = Robustly Optimized BERT。审核场景本质是分类任务（涉黄/涉政/正常），不需要生成。BERT 类双向注意力对短文本分类天然占优，且：
① 模型小（base 110M / large 340M），可本地部署 ② 推理快（一次 forward 出结果）③ 微调数据需求少（千级样本即可）④ 解释性强（可看 attention map）。
对生成式审核（带修改建议）才需要 Decoder-Only（如 Qwen3Guard）。
10预训练 vs 微调 vs RAG 选型

10.1 预训练做什么

用海量无标注文本（万亿 token 级别），让模型学会语言的统计规律。GPT-3 在 300B token 上训练，Qwen3 在 36T token 上训练。这是大模型"涌现能力"的来源。

11上下文窗口与 KV Cache（推理性能基础）

11.1 上下文窗口是什么

模型一次能处理的最大 token 数。Qwen3 默认 128K，DeepSeek-V3 128K。决定能塞多长的文档/对话历史。
注意：上下文越长，推理成本越高（attention 是 O(n²)，KV Cache 是 O(n)），不是越长越好。

11.2 KV Cache 直觉

生成式模型一个 token 一个 token 出。每生成一个 token 要做一次完整 attention——但前面 t-1 个 token 的 K 和 V 在每一步都一样，重复算就浪费。KV Cache = 把已经算过的 K/V 存下来，下一步直接用。

11.3 内存计算（面试可能考）

KV Cache 内存 = 2 × num_layers × num_heads × d_head × seq_len × dtype × batch

例：Qwen3-7B（32 层，32 头，d_head=128，FP16），batch=1，seq=8K：
2 × 32 × 32 × 128 × 8192 × 2 byte ≈ 4.3 GB。
单条 8K 上下文就吃 4G 显存——为什么大模型推理这么吃显存的核心原因。

引出 Phase 4 重点：MQA / GQA（多个头共享一组 KV）、PagedAttention（vLLM 核心）、量化 KV Cache，都是为了压缩这个数字。

11.5 MoE 混合专家（2026 必考）

2026 主流大模型（DeepSeek-V3 671B、Qwen3-235B-A22B、Mixtral 8x22B、GPT-4 系列推测、Claude 4.5）都用 MoE 架构。如果不会讲 MoE，P8 面试基本卡死。

11.5.1 一句话原理

把 FFN 层换成 N 个并列的 Expert（专家），每个 token 经过一个 Router（门控）选 Top-K 个专家激活，剩下的不参与计算。
DeepSeek-V3：671B 总参数，激活 37B（Top-8 of 256）→ 推理算力 ≈ 37B 稠密模型，但效果接近 671B。

11.5.2 计算流程（伪代码）

# 每个 token 单独路由，不是整个 batch 一起
for token in sequence:
    gate_logits = Router(hidden)        # [num_experts]
    top_k_idx = topk(gate_logits, k=2)   # 选 Top-2
    top_k_w = softmax(top_k_idx)         # 加权
    out = sum(top_k_w[i] * Expert[idx](hidden) for i, idx in top_k)

11.5.3 三大工程难点（面试连环问）

  1. 负载均衡：朴素训练会出现少数专家被狂选、大部分专家闲置 → 加 auxiliary loss 或 DeepSeek 的 auxiliary-loss-free（用偏置项软调）。
  2. 显存爆炸：671B 总参数即使激活 37B，显存还是要装下全部专家 → 用 EP（Expert Parallel）把不同专家分到不同 GPU。
  3. 训练不稳定：路由抖动导致梯度噪声大 → DeepSeekMoE 用「细粒度专家 + 共享专家」（256 个细专家 + 8 个共享专家）改善。

11.5.4 稠密 vs MoE 选型

维度稠密（Llama3-70B）MoE（DeepSeek-V3）
训练成本固定同效果下省 5-10x FLOPs
推理显存140GB1342GB（全专家加载）
推理算力140G FLOPs/token74G FLOPs/token
部署难度低高（需 EP + 高带宽互联）
适合场景单卡 / 小集群大集群 / 云推理

11.5.5 面试三段位答法

P7 答：知道 MoE 是混合专家，DeepSeek 用了。
P8 答：「MoE 把 FFN 改成 N 个 Expert + Router 选 Top-K。DeepSeek-V3 671B 总参激活 37B，是稀疏激活。三大难点：负载均衡（aux loss）、显存（EP 并行）、训练稳定（细粒度+共享专家）。和稠密相比同效果省 5-10x 训练 FLOPs，但推理需大集群 + 高速互联。」
P9 答：「MoE 是计算-显存 trade-off：用更多显存换更少算力。选稠密还是 MoE 看部署形态——单机/边缘 → 稠密；大集群云推理 → MoE。2026 趋势是 MoE 通用化（GPT-4/Claude 4.5/Gemini 都是 MoE 推测），稠密只在端侧/小模型场景。审核业务 70B 以下选稠密（Qwen3-8B），疑难复核可调用 DeepSeek-V3 MoE API。」
易踩坑：① 误以为 MoE 推理省显存——错，全专家都要在显存里；② 把 K（每 token 选几个专家）和 N（总专家数）搞混；③ 单卡跑 MoE 模型不开 EP，专家来回切换比稠密还慢。

13CoT / Self-Consistency / ToT

13.1 CoT（Chain of Thought）

核心发现：让模型在给出最终答案前先"写出推理过程"，准确率显著提升。
触发方式：① Zero-shot CoT：在 prompt 里加 "Let's think step by step" ② Few-shot CoT：示例里展示推理过程

13.2 Self-Consistency

同一问题让模型答 N 次（temperature>0），投票取最多的答案。GSM8K 上 +18% 准确率。代价：N 倍 token 成本。

13.3 ToT（Tree of Thoughts）

把推理建模成树搜索，每步生成多个候选思路，评分后选最优分支继续。适合"24 点"、密码破解这类需要回溯的题。

15Test-Time Compute（测试时计算）

15.1 概念

传统观点：算力都花在训练，推理越快越好。新观点（OpenAI o1 后）：推理时多花算力可换取更高质量，类似人类"想得久答得对"。

15.2 三种增加推理算力的方式

  1. 更长思维链（推理模型默认做的）：让模型多想几千 token 再输出答案
  2. 多次采样投票（Self-Consistency / Best-of-N）：算 N 次取最优
  3. 搜索 + 验证（ToT、PRM 引导搜索）：用 Process Reward Model 评每一步，剪枝

15.3 经济学视角

训练一次模型 = 一次性高成本。推理时 = 每次调用都付费。Test-Time Compute 让"小模型 + 多算"可能赢"大模型 + 一次"——这是 2026 行业的关键变化。审核场景应用：对疑难案例多算几次比一次过更准。

🎯 Phase 1 高频面试题（20 道，含答案要点）

1. 用一句话解释 Self-Attention
把序列里每个位置的特征，按它和其他位置的"相关度"加权聚合，让任意两个位置可以直接交互。
2. Q、K、V 分别是什么？为什么不直接用 X？
Q/K/V 都是 X 的线性投影：Q = X·W_Q（问询）、K = X·W_K（被检索）、V = X·W_V（信息载体）。直接用 X 会限制表达力（Q 和 K 必须长得一样），分开投影让模型能学三种不同视角。
3. 为什么 Attention 公式里要除 √d_k？
d_k 越大，q·k 内积方差越大（≈ d_k），softmax 在大方差下变陡峭，几乎只激活最大值，其他位置梯度为 0，模型学不动。除 √d_k 把方差拉回 1，softmax 平缓，梯度健康。
4. Multi-Head 比 Single-Head 好在哪？
单头只能学一种关注模式（如句法）；多头可以并行学多种（句法/语义/位置/共指等），最后拼起来。代价是参数量不变（d_model 切成 h 份），所以"几乎免费的多视角"。
5. BERT 和 GPT 的本质差异？
① 结构：BERT Encoder-only 双向 / GPT Decoder-only 因果 ② 训练：BERT MLM（遮词预测）/ GPT NTP（next token）③ 用法：BERT 微调下游 / GPT prompt 直用 ④ 现状：BERT 仍主导分类抽取，GPT 主导生成。
6. 为什么现在主流大模型都用 Decoder-Only？
① 训练目标统一（NTP 一招通吃）② 工程简单（一种 attention）③ Scaling Law 上 Decoder-Only 表现最佳 ④ Few-shot/in-context learning 能力强 ⑤ 长上下文友好。Encoder-Only 仅在专门 NLU 任务（分类抽取）有优势。
7. 位置编码为什么必要？演进路线？
Self-Attention 位置无关，必须额外注入位置信息。1代正弦/余弦绝对位置 → 2代可学习 embedding → 3代相对位置（T5 bias、ALiBi、RoPE）。2024+ 主流是 RoPE，外推性最好。
8. RoPE 比之前的位置编码强在哪？
① 用旋转矩阵把位置编进 Q/K，<q_m, k_n> 自然只依赖 m-n（相对位置）② 训练 4K 推理 32K 也能用（外推性好）③ 不增加参数 ④ 配合 NTK/YaRN 可达 128K+。
9. LayerNorm 和 BatchNorm 区别？为什么 Transformer 用前者？
BN 跨样本归一化（依赖 batch_size，序列变长时不便），LN 跨特征归一化（与 batch 解耦）。Transformer 序列任务变长且推理 batch=1 常见，必须用 LN。
10. Pre-Norm 和 Post-Norm 选哪个？为什么？
2024+ 主流 Pre-Norm（在 attention/FFN 前 norm）。原因：训练更稳定，深层（>100 层）也能收敛。Post-Norm 表达更强但容易梯度爆炸。Llama 等还把 LN 换成 RMSNorm（去掉减均值，更快）。
11. 残差连接的作用？没有它会怎样？
① 梯度高速公路（缓解深层梯度消失）② 恒等学习（最坏情况是不变，不会更差）③ 是堆深层的前提。没有残差连接，10 层以上 Transformer 训不动。
12. KV Cache 是什么？为什么必要？
生成式模型一次出一个 token，每步要做完整 attention，但前面 t-1 个 token 的 K/V 不变。把它们缓存下来下次直接用，就是 KV Cache。不做就 O(n²) 重复计算，慢且贵。
13. KV Cache 占多大显存？怎么算？
公式：2 × num_layers × num_heads × d_head × seq_len × dtype_bytes × batch。Qwen3-7B（32层32头d=128 FP16）batch=1 seq=8K → ≈4.3GB。压缩方案：MQA/GQA、PagedAttention、INT8 KV 量化。
14. 上下文窗口怎么从 4K 扩到 128K？
① RoPE base 频率插值（PI）② NTK-aware（高频不变低频拉伸）③ YaRN（智能分段缩放）④ 长度外推训练（先短后长继续训练）⑤ 用 Sliding Window / Sparse Attention 让 attention 不再 O(n²)。Qwen2.5、Llama3 都靠组合方案。
15. 幻觉是什么？怎么缓解？
大模型自信输出错误信息。成因：训练目标偏向"流畅"非"正确"、训练数据噪声、知识过时、长尾不足、解码随机。缓解：RAG 注入事实、降低 temperature、引用验证、self-consistency 投票、对齐训练。
16. CoT 为什么有效？
把"一步出答案"变成"多步小推理"，每步只解决子问题，复杂度大大降低。本质是把模型推理空间从单 token 扩展到多 token，让 attention 在中间步骤分摊计算。GSM8K 上 +30% 准确率。
17. CoT、Self-Consistency、ToT 的差别？
CoT：单条思维链（1×成本）。Self-Consistency：多条思维链投票（5-10×）。ToT：树搜索 + 评分剪枝（20-50×）。成本与可靠性递增。审核疑难案例可用 Self-Consistency。
18. o1 / R1 这类"推理模型"和普通大模型的区别？
普通模型靠 prompt 触发 CoT；推理模型把"长 CoT 思考"内化到权重里，生成时自动产生超长思维链。训练核心是大规模 RL（DeepSeek-R1 用 GRPO）。强项：数学/代码/逻辑；弱项：闲聊、风格化创作、低延迟场景。
19. DeepSeek-R1 的 GRPO 是什么？比 PPO 强在哪？
GRPO = Group Relative Policy Optimization。同一 prompt 采样一组回答（如 8 条），组内归一化算 advantage，不需要 critic 模型。比 PPO 省一半显存且训练更稳。是 R1 涌现长 CoT 的关键。
20. Test-Time Compute 是什么？为什么 2026 这么火？
推理时多花算力换更高质量（更长 CoT、多次采样、搜索验证）。火起来因为：① o1 实证有效 ② 训练 Scaling 趋缓，推理 Scaling 是新维度 ③ 让小模型 + 多算可以打败大模型 + 一次。审核场景：疑难案例多算几次比一次过更准。
