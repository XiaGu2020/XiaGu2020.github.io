# Stock Advisor 开发记录

> 一个网文后端写量化工具的笨办法。

## 起因

主业做内容审核，每天面对的是规则、模型、误伤率。下班想换个脑子，又不想白费时间，于是把"个人投资决策"这件事工程化——做一个自己用的 A 股分析工具：**Stock Advisor**。

## 技术选型

| 模块 | 选型 | 理由 |
|---|---|---|
| 后端 | Node.js | 起项目快，调第三方接口爽 |
| 前端 | Vue3 + Vite | 熟练，组件生态成熟 |
| 数据源 | 第三方行情 API | 免费 + 稳定优先 |
| 部署 | 本地为主 | 自用，不上云省事 |

## 踩过的坑

### 1. 不要用 PE 一刀切

2026-05 大族激光的复盘教训：当时 PE 91/203 直接判"高估不能买"，结果 6 个交易日涨 +24%。**趋势股的核心是资金 + 情绪，PE 是后验指标**。

判断短线建仓的核心三要素改为：
1. 主力资金流入持续性（5/10/20 日共振）
2. 筹码集中度变化（下降 = 机构吸筹）
3. 板块共振 + 领涨股效应

### 2. 数据源缓存

行情数据高频但变化慢，加 Redis 风格的内存缓存，TTL 5 秒，省接口费。
